### 1. v2.0.0-beta.21 休息機制 bug 修復後重新測試

**已發布**：https://github.com/lingoota/timer-app/releases/tag/v2.0.0-beta.21
**beta.21 修的（女兒回報）**：
- 計時到了 → 立刻開始算休息（不等按掉警報；解決「離開休息完回來按結束才開始倒數」）
- 重置/迷你結束只在「進行中/暫停中」才休息（修「完成後按重置被要求休息整個選定時間」誤觸發）

重新驗證：
- [ ] **完成後立刻休息**：計時跑完 → 不按警報直接離開 → 回來時休息已倒數（甚至已結束）
- [ ] **完成後按重置**：跑完按重置 → 不會「又被要求休息整段時間」（休息只算一次、從完成起算）
- [ ] **未開始按重置**：選好時間沒開始就按重置 → 完全不要求休息
- [ ] **真正提前結束**：跑到一半按重置/迷你結束 → 按實際用掉時間休息
- [ ] **1:1 體感**：用 30 分要休 30 分，實際是否合理（如太硬再評估比例/上下限）

沿用 beta.19 待驗證項（beta.20 已含 beta.19 所有修復）：
- [ ] **主要場景**：開瀏覽器全螢幕 YouTube，計時 20 分鐘 → 警報準時、迷你視窗顯示準時
- [ ] 開始 → 計時運作 + 進入迷你模式正常
- [ ] 暫停 → 計時暫停，再點開始 → 從剩餘時間繼續
- [ ] 重置 → 計時歸零 + 按實際用掉時間進入休息
- [ ] 迷你視窗暫停/繼續 → 主視窗 UI 同步、計時準確
- [ ] 迷你視窗結束 → 回主視窗、歸零 + 按實際用掉時間進入休息
- [ ] 自然完成 → 從迷你模式恢復、置頂、警報音響、Firebase 寫記錄、家長監控歸零
- [ ] 警報音效循環第 2 次強制置頂
- [ ] 暫停後改時間 → 視為新計時（不被當成 resume）
