## 🔜 下次開發項目

### 🏗️ 程式碼重構（模組化）

> **目的**：renderer.js 已膨脹到 2,671 行，所有功能混在一起。拆分後每次修改只需讀取目標模組，省 token、更精準、更好維護。

**現況問題**：
| 檔案 | 行數 | 狀態 |
|------|------|------|
| renderer.js | 2,671 | 過大，需拆分 |
| main.js | 933 | 偏大，次優先 |

**renderer.js 拆分方案**：
```
src/
├── renderer.js       → 主入口（初始化、事件綁定、共享狀態）~300 行
├── timer.js          → 計時系統（start/pause/reset/update/sound）~400 行
├── users.js          → 用戶系統（切換、身份綁定、家長模式）~300 行
├── charts.js         → 統計圖表（三種圖 + 匯出）~500 行
├── reminders.js      → 閒置提醒 + 冷卻期 UI ~200 行
├── updater-ui.js     → 更新相關 UI ~100 行
```

**關鍵注意事項**：
- 共享狀態（`state` 物件、DOM 元素引用）需設計傳遞方式
- Electron renderer 環境不支援 ES modules（需用全域變數或 window 掛載）
- 拆分後需完整測試所有功能
- 純重構，不改功能邏輯

**執行順序**：
1. 先拆最獨立的模組（charts.js — 圖表跟核心邏輯耦合最低）
2. 再拆 users.js（用戶系統）
3. 再拆 timer.js（計時核心）
4. 最後拆 reminders.js + updater-ui.js
5. 每拆一個就測試一次，確認無 regression
