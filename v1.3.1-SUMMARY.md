## 🚀 **新功能概覽**

### 1️⃣ **計時完成窗口置頂**
**目的**: 確保用戶不會錯過計時結束提醒

**實現方式**:
- 計時結束時發送 IPC 消息 `timer-completed-show-window`
- 主進程自動將窗口置頂並獲得焦點
- 從迷你模式自動恢復到正常模式
- 5秒後自動取消置頂狀態

**用戶體驗**:
- ✅ 無論用戶在使用什麼應用程式，計時結束時都會看到提醒
- ✅ 自動從迷你模式恢復，無需手動操作
- ✅ 短暫置頂後自動恢復，不會永久干擾其他應用

### 2️⃣ **智能5分鐘計時冷卻期**
**目的**: 防止過度使用，鼓勵適度休息

**邏輯設計**:
- 計時完成時 → **可立即重新開始計時**
- 手動停止警報後 → **才開始5分鐘冷卻期**
- 冷卻期間開始按鈕顯示倒計時
- 每秒更新剩餘時間顯示

**用戶體驗**:
- ✅ 靈活性：用戶可以選擇立即重新計時或停止警報休息
- ✅ 清晰反饋：按鈕實時顯示「冷卻中 (X分鐘)」
- ✅ 智能提醒：冷卻結束時顯示「可以重新開始計時了！」

### 3️⃣ **循環警報系統**
**目的**: 確保用戶在忙碌時不會錯過提醒

**循環邏輯**:
- 計時結束 → 1分鐘警報音效
- 1分鐘後自動停止 → 30秒靜音
- 30秒後重新響起 → 持續循環
- 手動停止可終止循環

**用戶體驗**:
- ✅ 持續提醒：即使用戶暫時無法處理，也會持續得到提醒
- ✅ 適度間隔：30秒間隔避免過於打擾
- ✅ 完全控制：隨時可手動停止循環

## 🔧 **技術實現亮點**

### **跨進程通信 (IPC)**
```javascript
// 渲染進程 → 主進程
window.api.send('timer-completed-show-window');

// 主進程處理
ipcMain.on('timer-completed-show-window', (event) => {
    mainWindow.show();
    mainWindow.focus();
    mainWindow.setAlwaysOnTop(true);
    // 5秒後取消置頂
});
```

### **狀態管理優化**
```javascript
const state = {
    lastCompletionTime: null,        // 冷卻期開始時間
    cooldownDuration: 5 * 60 * 1000, // 5分鐘冷卻
    cooldownUpdateInterval: null,     // 倒計時更新器
    alarmCycleActive: false,         // 循環警報狀態
    alarmCycleTimeout: null,         // 循環定時器
};
```

### **邏輯分離設計**
- **自動停止警報**: 不觸發冷卻期，支持循環
- **手動停止警報**: 觸發冷卻期，終止循環
- **清晰的狀態轉換**: 防止邏輯衝突
