## ⚙️ 自動配置指示

### 1. 更新 Claude Code 配置檔
請協助我設定以下配置檔案：

**專案配置** (`.claude/settings.json`)：
- 設定適合新手的工具權限
- 啟用 TodoRead 和 TodoWrite 工具
- 設定中文為預設語言環境

**專案說明檔** (`CLAUDE.md`)：
- 建立專案概覽文件
- 記錄常用指令和工作流程
- 說明 WSL 環境特殊注意事項

### ⚡ 專用提示詞設定
建立以下快捷指令，放在 `.claude/commands/` 資料夾中：

#### 📋 **【繼續】** (`/繼續`)
自動執行：
1. 讀取 `todo.md` 了解當前進度
2. 檢查最近的 commit 和檔案變更
3. 列出專案現況摘要：
   - ✅ 已完成的項目
   - 🚧 進行中的工作
   - 📝 下一步要做的事情
   - 💡 需要注意的重點
4. 詢問要從哪個項目開始繼續

#### 🔄 **【狀態】** (`/狀態`)
快速檢視：
- 專案整體進度
- Git 狀態（未提交的變更）
- 最近的工作重點
- 待解決的問題

#### ✅ **【完成】** (`/完成`)
標記當前工作為完成：
1. 更新 todo.md
2. 更新 專案概況.md 的進度
3. 記錄學到的重點
4. 準備下一個任務

#### 🚧 **【暫停】** (`/暫停`)
暫停當前工作：
1. 記錄目前進度
2. 更新 todo.md 的暫停狀態
3. 更新 專案概況.md 的當前狀態
4. 記錄下次繼續的提醒

#### 🚀 **【推送】** (`/推送`)
一鍵完成所有更新和推送：
1. 檢查 Git 狀態和未提交的變更
2. 自動執行 `git add .`
3. 要求輸入繁體中文 commit message
4. 執行 `git commit` 和 `git push`
5. 自動更新 專案概況.md 記錄推送時間和里程碑
6. 更新 todo.md 標記版本里程碑

### 2. 建立專案 Todo List
每次開始新專案時，請：

1. **建立 `todo.md` 檔案**，包含：
   - 專案目標和功能需求
   - 技術學習目標
   - 階段性里程碑
   - 下次繼續的提醒事項

2. **建立 `專案概況.md` 專案摘要文檔**，包含：
   - 📋 專案基本資訊（目標、技術棧、開始日期）
   - 🎯 核心功能清單和完成狀態
   - 📈 進度概覽（整體完成度、當前階段）
   - 🏆 重要里程碑和推送記錄
   - 💡 重要學習筆記
   - ⚡ 可用的專用指令列表
   - 🔄 最後更新時間和狀態

3. **在工作過程中持續更新 todo list**：
   - ✅ 完成的項目標記為已完成
   - 🚧 進行中的項目標記狀態
   - 📝 新增發現的待辦事項
   - 💡 記錄學到的重要概念

4. **每次結束工作時**：
   - 更新當前進度
   - 記錄下次要繼續的具體步驟
   - 標注需要特別注意的地方
   - **使用 `/推送` 自動更新所有狀態並推送到 GitHub**
