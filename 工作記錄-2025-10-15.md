### 3. 📦 應用程式封裝

#### 挑戰與解決
**遇到的問題**:
1. **符號連結權限錯誤**
   - 錯誤：`Cannot create symbolic link : 用戶端沒有所需的特殊權限`
   - 原因：Windows 需要管理員權限建立符號連結
   - 工具：`winCodeSign-2.6.0.7z` 解壓縮失敗

2. **代碼簽名問題**
   - electron-builder 嘗試使用代碼簽名工具
   - 即使設定 `sign: false` 仍然下載簽名工具

**解決方案**:
1. 設定環境變數禁用簽名自動偵測
   ```bash
   set CSC_IDENTITY_AUTO_DISCOVERY=false
   ```

2. 調整建置目標
   - 原始：`portable` 格式（需要簽名）
   - 最終：`dir` 格式（無需簽名）

3. 更新 `package.json` 配置
   ```json
   {
     "win": {
       "target": "dir",
       "sign": null,
       "verifyUpdateCodeSignature": false
     }
   }
   ```
