
### 💡 技術方案討論

#### 議題一：資料儲存方式

**需求分析**:
- ❌ 不想用 Google Sheets（有延遲、限制多）
- ✅ 要雲端同步（品瑜和品榕的資料要同步）
- ✅ 簡單易用（不要太複雜的設定）
- ✅ 免費方案（個人使用不想付費）

**考慮的方案**:

1. **Firebase Firestore** ⭐ (最終選擇)
   - ✅ 完全免費（每天 50,000 次讀取、20,000 次寫入）
   - ✅ 即時同步（不再有延遲）
   - ✅ 超級簡單（幾行程式碼就能用）
   - ✅ Google 官方支援（穩定可靠）
   - ✅ 有離線支援
   - 程式碼範例：
     ```javascript
     // 寫入資料（只要 3 行）
     const db = firebase.firestore();
     await db.collection('users').doc('pinyu').set({ todayUsage: 120 });

     // 讀取資料（只要 2 行）
     const doc = await db.collection('users').doc('pinyu').get();
     console.log(doc.data());
     ```

2. **JSON 檔案 + 雲端資料夾同步** (通用版規劃)
   - ✅ 完全免費（利用現有 Google Drive / Dropbox）
   - ✅ 完全掌控（資料存在自己雲端）
   - ❌ 不是即時同步（要等雲端軟體同步）
   - ❌ 可能有衝突（兩台電腦同時寫入）

3. **Supabase** (PostgreSQL)
   - ✅ 免費方案大方（500MB、50,000 次請求/月）
   - ⚠️ 設定較複雜
   - ⚠️ 需要學習 SQL

4. **MongoDB Atlas**
   - ✅ 免費 512MB
   - ⚠️ 設定較複雜
   - ❌ 沒有即時同步

**決策**: **Firebase Firestore**

**理由**:
1. 最簡單 - 20 分鐘就能整合完成
2. 最穩定 - Google 的服務，不會掛
3. 最適合需求 - 即時同步、完全免費
4. 使用量絕對不會超過免費額度

---

#### 議題二：應用程式自動更新

**需求分析**:
- ❌ 不想手動下載、置換（目前的痛點）
- ✅ 自動推送更新（輕鬆方便）
- ✅ 可以是安裝版或免安裝版

**考慮的方案**:

1. **electron-updater + GitHub Releases** ⭐ (最終選擇)
   - ✅ 完全免費（使用 GitHub Releases）
   - ✅ 自動更新（應用程式啟動時自動檢查）
   - ✅ 官方支援（Electron 官方推薦）
   - ✅ 下載進度顯示
   - ⚠️ 需要改成安裝版（免安裝版無法自動更新）
   - 運作方式：
     1. 發布新版本 → 上傳到 GitHub Releases
     2. 應用程式啟動 → 自動檢查更新
     3. 發現新版本 → 自動下載並安裝
     4. 重新啟動 → 完成更新

2. **自動檢查更新 + 一鍵下載** (半自動)
   - ✅ 保持免安裝版
   - ⚠️ 還是需要手動置換
   - 改善有限

3. **雲端硬碟共享 + 版本檢查**
   - ✅ 最簡單（上傳到 Google Drive）
   - ❌ 還是手動下載
   - 改善有限

**決策**: **改成安裝版 + electron-updater**

**理由**:
1. 一次設定，永遠方便（女兒只要安裝一次）
2. 標準做法（Chrome、VS Code、Discord 都這樣）
3. 使用者體驗最好（自動完成）

## 2026-03-03：計時提醒 + 開機自啟動 + 身份綁定功能

### 📋 使用者需求
```
品瑜和品榕常常忘記計時，甚至忘記開啟程式，導致被媽媽懲罰。
需要：(1) 安裝時綁定身份 (2) 小孩身份開機自動啟動 (3) 偵測到使用電腦但沒計時就彈出提醒。
```

### ✅ 處理成果
- **身份綁定**：首次啟動顯示身份選擇（品瑜/品榕/家長模式），選擇後永久綁定到 `localStorage('installedUser')`，可在設定中切換
- **開機自啟動**：品瑜/品榕選擇後自動設定 `app.setLoginItemSettings({ openAtLogin: true })`，家長模式不啟用
- **閒置偵測提醒**：每 5 秒檢查 `powerMonitor.getSystemIdleTime()`，使用電腦超過 60 秒未計時就彈出提醒
- **提醒對話框**：「開始計時」和「稍後提醒（5分鐘冷卻）」兩個按鈕
- 修改檔案：`main.js`、`src/renderer.js`、`src/index.html`、`src/styles.css`
